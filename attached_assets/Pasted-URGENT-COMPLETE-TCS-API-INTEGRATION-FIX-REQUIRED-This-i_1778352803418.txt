Developer is currently using the authentication flow incorrectly.

TCS APIs use TWO TOKENS:

1. Bearer Token
   → Used in Authorization Header

2. ECOM Access Token
   → Returned after Authentication API
   → Used INSIDE booking JSON payload

Current system is mixing both tokens incorrectly.

That is why errors are coming:

❌ Invalid Bearer token
❌ Invalid access token
❌ Mismatch configuration
❌ 401 Unauthorized
❌ 405 Method Not Allowed
❌ shipmentinfo.weightinkg invalid
